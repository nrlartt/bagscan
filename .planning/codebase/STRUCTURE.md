# Codebase Structure

**Analysis Date:** 2026-03-09

## Directory Layout

```
bagscanai/
├── .codex/                  # GSD workflows, templates, skills, local metadata
├── prisma/                  # Prisma schema and SQL migrations
├── public/                  # Static assets and runtime-uploaded files
├── scripts/                 # Utility scripts
├── src/                     # Application code (Next.js app, libs, components)
├── package.json             # Scripts and dependency manifest
├── tsconfig.json            # TypeScript compiler config
├── next.config.ts           # Next.js runtime config
└── README.md                # Project usage and setup docs
```

## Directory Purposes

**`src/app/`:**
- Purpose: Next.js App Router pages and API endpoints.
- Contains: page components (`page.tsx`), nested routes (`launch`, `token/[mint]`), API route handlers (`api/**/route.ts`).
- Key files: `src/app/page.tsx`, `src/app/layout.tsx`, `src/app/launch/page.tsx`.
- Subdirectories: `api/`, `token/`, `launch/`, `alpha/`, `about/`.

**`src/components/`:**
- Purpose: Reusable UI components and provider wiring.
- Contains: BagScan UI widgets and app-level providers.
- Key files: `src/components/Providers.tsx`, `src/components/bagscan/TokenCard.tsx`.
- Subdirectories: `bagscan/`, `launch/`.

**`src/lib/`:**
- Purpose: Non-UI business logic, client wrappers, mappers, validation, and utility modules.
- Contains: Bags client wrappers, sync engine, alpha engine, solana helpers, db wrapper.
- Key files: `src/lib/bags/client.ts`, `src/lib/sync/index.ts`, `src/lib/db/index.ts`.
- Subdirectories: `bags/`, `sync/`, `alpha/`, `solana/`, `validators/`, `xquik/`, `utils/`, `db/`.

**`prisma/`:**
- Purpose: Data model and migration source.
- Contains: `schema.prisma`, migration SQL files.
- Key files: `prisma/schema.prisma`, `prisma/migrations/0001_init/migration.sql`.

**`public/`:**
- Purpose: Browser-served static files.
- Contains: icons and uploaded files.
- Key files: `public/favicon.svg`, `public/uploads/*` (runtime-created).

**`.codex/`:**
- Purpose: Local Codex/GSD workflow assets.
- Contains: `get-shit-done/`, `skills/`, local manifests.

## Key File Locations

**Entry Points:**
- `src/app/layout.tsx` - Global app shell, providers, nav/footer.
- `src/app/page.tsx` - Main discovery UI.
- `src/app/api/tokens/route.ts` - Main token list/search backend entry.

**Configuration:**
- `package.json` - Scripts and dependency list.
- `tsconfig.json` - strict mode, path alias config.
- `eslint.config.mjs` - linting presets.
- `next.config.ts` - Next.js config surface (currently minimal).
- `.env.example` - expected environment variable keys.

**Core Logic:**
- `src/lib/sync/index.ts` - token feed aggregation/caching.
- `src/lib/bags/client.ts` - external Bags/DexScreener/Helius calls.
- `src/lib/alpha/engine.ts` - alpha signal generation.
- `src/app/api/launch/*` - launch orchestration routes.

**Testing and Quality:**
- No dedicated `tests/` directory currently.
- Quality tooling from scripts: `npm run lint`, `npx tsc --noEmit`.

**Documentation:**
- `README.md` - setup and product overview.
- `.planning/codebase/*.md` - generated codebase map docs.

## Naming Conventions

**Files:**
- `PascalCase.tsx` for many React components in `src/components/bagscan/`.
- `page.tsx` and `route.ts` reserved by App Router conventions.
- `index.ts` used in several lib directories as module entry.

**Directories:**
- Mostly lowercase and route-oriented (`launch`, `alpha`, `api`, `bags`, `sync`).
- Dynamic route directories use bracket notation (`[mint]`).

**Special Patterns:**
- `@/*` alias maps to `src/*`.
- API endpoints are defined by `src/app/api/**/route.ts`.

## Where to Add New Code

**New API endpoint:**
- Implementation: `src/app/api/<feature>/route.ts`.
- Validation schema: `src/lib/validators/index.ts` (or dedicated validator module).
- External API helpers: `src/lib/<domain>/client.ts`.

**New UI feature:**
- Page-level route: `src/app/<route>/page.tsx`.
- Shared UI component: `src/components/bagscan/` or `src/components/<feature>/`.
- Shared utility: `src/lib/utils/` or relevant domain folder.

**New persistence model:**
- Add model in `prisma/schema.prisma`.
- Create migration in `prisma/migrations/`.
- Access via `src/lib/db/index.ts` + consuming services.

## Special Directories

**`.next/`:**
- Purpose: Next.js build and dev artifacts.
- Source: generated by `next dev/build`.
- Committed: no (ignored build output).

**`node_modules/`:**
- Purpose: installed dependencies.
- Source: generated by npm install.
- Committed: no.

**`public/uploads/`:**
- Purpose: runtime image storage fallback for launch metadata.
- Source: generated by upload routes at runtime.
- Committed: generally no (runtime artifact).
